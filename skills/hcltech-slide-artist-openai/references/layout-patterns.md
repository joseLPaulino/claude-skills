# HCLTech Slide Layout Patterns

Use these reusable central-visual patterns to reduce prompt ambiguity.

## Three-Layer Operating Model

Use for strategy, adoption model, governance model, or delivery model briefs.

- Layout: three stacked horizontal layers or three vertical cards with a clear outcome band.
- Risks: too much text per layer; weak relationship between layers and outcome.
- Prompt snippet: "Create three stacked rounded cards connected by thin gradient lines, labelled with the exact layer names, flowing into a concise outcome panel on the right."

## Maturity Curve

Use for transformation journeys, current-to-future state, capability growth, or adoption stages.

- Layout: rising curve across four or five milestone nodes.
- Risks: dense milestone text; decorative charts that obscure the key progression.
- Prompt snippet: "Show a clean rising maturity curve with large milestone nodes, short exact labels, and a final target-state marker in the blue-to-purple gradient."

## Capability Map

Use for domains, services, operating capabilities, or platform components.

- Layout: central capability with surrounding grouped cards or a two-by-three grid.
- Risks: too many small boxes; unclear grouping.
- Prompt snippet: "Build a structured capability map with six large cards grouped into two rows, each card containing one exact label and a simple icon."

## KPI Dashboard

Use for metrics, outcomes, performance, or executive status summaries.

- Layout: three to five large metric cards with a small trend or gauge visual.
- Risks: tiny numbers; invented metrics; visual noise.
- Prompt snippet: "Create a clean executive KPI dashboard with four large metric cards, oversized numbers, minimal labels, and subtle gradient accents."

## Roadmap Timeline

Use for phased delivery, implementation plans, migration, or sequencing.

- Layout: left-to-right timeline with three to five phases.
- Risks: cramped phase descriptions; weak visual distinction between completed and future phases.
- Prompt snippet: "Show a horizontal roadmap timeline with large phase markers, short exact labels, and a clear progression from discovery to scale."

## Hub-And-Spoke Ecosystem

Use for partnerships, platforms, stakeholder networks, or system ecosystems.

- Layout: central hub with five to seven surrounding nodes connected by thin lines.
- Risks: too many spokes; illegible node labels; arrows that imply the wrong direction.
- Prompt snippet: "Place the core concept as a central rounded hub with surrounding ecosystem nodes, each with a short exact label and simple iconography. Use neutral connector lines without arrowheads unless the brief explicitly describes flow direction."
